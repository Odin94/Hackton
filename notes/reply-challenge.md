# Reply Challenge — Hard Facts

## Host
- Reply Challenge at TUM (Technical University of Munich).

## Mandate
- Build an autonomous AI agent.

## Theme (from challenge giver)
- Create a **campus copilot suite**.
- Emphasis on **one agent system** (not a swarm of specialized agents).
- Target: something that supports everyday student life.

## Framing (expected structure)
1. Identify the problem.
2. Design the agent.
3. Show the impact.

## Bonus
- Bonus points for using **cognee** (memory / knowledge-graph framework).

## Resources provided
- AWS access.

## Rating criteria (25 % each)

### 1. Innovation / ambition
- Uniqueness / creativity
- Ambition / USP
- Concept execution
- AI integration

### 2. UI / UX
- Aesthetics and polish
- Usability
- Workflow and navigation

### 3. Overall quality
- Functionality
- AI effectiveness
- Critical features
- Scoping

### 4. Presentation quality
- Preparation
- Communication
- Overall delivery
