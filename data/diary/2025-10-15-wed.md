**Morning — Informatik exercise session**

Lost 30 minutes to an @Override bug. We were implementing a class hierarchy in Java — Shape as the base, Circle and Rectangle as subclasses. I wrote the area() method in Circle without the @Override annotation. Java compiled it as a new method instead of an override, so when I called area() through a Shape reference it dispatched to the base class version and returned 0.

I stared at this for longer than I should have. Eventually a TA spotted it in 10 seconds.

Afterwards I mentioned it to TumTum. It said this is one of the most common Java gotchas for people coming from Python, because Python doesn't require (or support) explicit override annotations — the method resolution just works. Java requires you to be explicit because it's statically dispatched and the compiler can catch mistakes if you tell it what you intend. "The annotation is documentation for the compiler." I actually liked that framing.
